# Contributing to Civils Agri Plant Website

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## Development Setup

1. Clone the repository
2. Install dependencies: `npm install`
3. Run development server: `npm run dev`
4. Make your changes in the `source` folder
5. Build for production: `npm run build`

## Code Style

- Follow the existing code style
- Use `.editorconfig` settings (supported by most editors)
- Use 2 spaces for indentation
- Keep HTML semantic and accessible
- Add ARIA labels where appropriate

## File Structure

- `source/` - Source files (edit these)
- `theme/` - Built files (generated, don't edit directly)
- `source/partials/` - Reusable HTML partials
- `source/scss/` - SCSS stylesheets

## Commit Messages

- Use clear, descriptive commit messages
- Reference issue numbers if applicable

## Pull Requests

1. Create a branch for your changes
2. Make your changes
3. Test thoroughly
4. Submit a pull request with a clear description

## Questions?

Contact the project maintainers or open an issue.
